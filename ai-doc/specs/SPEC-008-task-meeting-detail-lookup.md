# SPEC-008 任務會議細節快速查找

狀態：Active
對應 DEV：DEV-008
建立日期：2026-06-06
承接：DEV-002 / DEV-006 / DEV-007

## 背景

每個任務的細節常在會議中被討論，但未來專案成員若只能進入紀錄庫翻整篇會議紀錄，查找成本太高。任務詳情應成為查找入口：使用者打開任務，就能看到這個任務曾在哪些會議或工作紀錄被討論、當時的片段、以及會議中產生的狀態變更。

## 目標

- 將任務詳情頁的「關聯紀錄」升級為「任務知識」入口。
- 優先顯示含有該任務 inline tag 的紀錄片段，而不是整篇紀錄摘要。
- 支援在目前任務範圍內搜尋備註、會議紀錄、工作紀錄與會議中任務變更。
- 點擊片段可回到原始紀錄，保留可追溯來源。
- 不新增資料模型，不依賴 AI 自動判斷。

## MVP

- 任務詳情頁顯示「任務知識」區塊。
- 已關聯紀錄依時間倒序顯示。
- 每筆紀錄抽出最多數個包含 `@[title](task:id)` 的段落片段。
- 若紀錄有 `record_task_links` 關聯但內容沒有該任務 tag，顯示「整篇關聯」摘要。
- 任務知識區塊提供搜尋框，搜尋範圍限於目前任務的備註與關聯紀錄片段。
- 會議中產生的任務狀態、日期、指派、移動等變更，因 DEV-007 已寫入會議紀錄內容，需可在同一區塊被看到與搜尋。

## 非範圍

- 不新增 migration。
- 不新增 meeting event table。
- 不做 AI 問答、AI 決議抽取或自動摘要。
- 不跨 board 搜尋未關聯任務。
- 不改 `KnowledgeRecord`、`record_task_links`、RAG token 格式。

## 使用者流程

1. 專案成員打開任務詳情。
2. 在任務知識區塊看到此任務的備註、關聯會議片段、工作紀錄片段與任務變更片段。
3. 輸入關鍵字，例如「延期」、「客戶規格」、「風險」。
4. 清單只保留目前任務相關的命中片段。
5. 點擊片段後開啟原始紀錄，供使用者閱讀完整上下文。

## 驗收標準

- 同一篇會議紀錄同時提到任務 A 與任務 B 時，任務 A 詳情只優先顯示任務 A 的片段，任務 B 詳情只優先顯示任務 B 的片段。
- 搜尋關鍵字只作用在目前任務的任務知識區塊。
- 沒有 task mention 但有結構關聯的紀錄仍可被看到，並標示為整篇關聯。
- DEV-007 寫入的「會議中任務變更」可在任務知識區塊顯示。
- 點擊紀錄片段可開啟原始紀錄。
- 不新增或修改資料模型。
