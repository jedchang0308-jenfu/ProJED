# SPEC-019：紀錄類型與會議流程層級重整

對應 DEV：DEV-019
父交付點：DEV-002 / DEV-005 / DEV-018
狀態：Implemented

## 背景

DEV-018 已把會議紀錄側欄改成 `速記 → AI整理 → 校稿 → 發布` 的防呆工作流，但 `會議紀錄` 與 `個人工作紀錄` 這兩個紀錄類型仍容易被使用者誤認為同一層流程按鈕。

本規格補足分層：

- `會議紀錄 / 個人工作紀錄` 是紀錄類型。
- `速記 / AI整理 / 校稿 / 發布` 是會議紀錄在會議模式中的流程階段。
- 個人工作紀錄不套用會議流程，只保留撰寫、存草稿、發布。

## 設計原則

介面分為三層：

1. Topbar：全域會議模式。
2. Record Sidebar：紀錄類型與面板工具。
3. Workflow：目前紀錄類型可執行的流程。

依 NN/g 的 system status 與 recognition rather than recall 原則，使用者應能直接從位置、形狀、icon、文案看出目前狀態與下一步，不需要記憶每顆按鈕的隱含意義。

## 功能需求

### 全域會議模式

- Topbar 顯示 `開始會議速記` 或 `離開會議`。
- `離開會議` 使用離開語意 icon，不使用箭頭。
- title 說明：離開不等於發布，未儲存時會詢問是否存草稿。

### 紀錄類型層

- 紀錄類型必須在開始撰寫前決定。
- 建立草稿後只顯示目前紀錄類型，不提供 `會議紀錄 / 個人工作紀錄` 切換按鈕。
- 若使用者選錯類型，應新增另一筆正確類型的紀錄，不在既有草稿上改類型。
- 非會議模式：
  - 空白狀態提示先選擇紀錄類型。
  - Header 新增按鈕文案為 `新增會議紀錄`、`新增個人工作紀錄`。
- 會議模式：
  - 顯示 `目前類型：會議紀錄（會議模式）`。
  - 不顯示可切換的 `個人工作紀錄` button。
  - 補充說明：`個人工作紀錄請離開會議模式後建立。`

### 會議紀錄流程

- 只在會議模式顯示 arrow workflow：
  - `速記`
  - `AI整理`
  - `校稿`
  - `發布`
- 每顆流程按鈕同時呈現階段與按下後果。
- `AI整理` 是選用動作，不是發布前必經門檻。

### 個人工作紀錄流程

- 個人工作紀錄不顯示 `AI整理 / 校稿`。
- 顯示簡單狀態列：
  - `撰寫中`
  - `已存草稿`
  - `已發布`
- 行動按鈕只保留：
  - `存草稿`
  - `發布工作紀錄`

## 非目標

- 不新增資料表或 migration。
- 不改 `KnowledgeRecordType`。
- 不改 RAG token 或 `record_task_links`。
- 不把個人工作紀錄套入會議 AI 流程。
- 不新增 BoardView 上方舊式會議操作列。

## 驗收標準

- 使用者能分辨紀錄類型與流程階段。
- 使用者在開始撰寫前決定紀錄類型，建立後不可在同一筆草稿上切換類型。
- 會議模式中只顯示會議紀錄流程。
- 個人工作紀錄不出現 `AI整理 / 校稿`。
- `新增會議紀錄` 不被誤解為 `開始會議速記`。
- `離開會議` 只在 Topbar。
- `收合面板` 只做面板收合。
- 1024x768 與 1440x950 無重疊、裁切、水平 overflow。
