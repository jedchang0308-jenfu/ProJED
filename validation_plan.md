# ProJED 功能驗證計畫 - 手機優化與行內編輯保護

## 1. 核心驗證清單 (Core Checklist)

### A. 手機端長按選單 (Mobile Long Press)
- [ ] **任務卡片長按**：在看板模式下長按任務卡片 500ms，應彈出右鍵選單。
- [ ] **待辦清單長按**：在卡片內的 Checklist 項目長按 500ms，應彈出右鍵選單。
- [ ] **群組標題長按**：在看板列標題（Column Header）長按 500ms，應彈出右鍵選單。
- [ ] **選單功能完整性**：選單內的「新增子任務」、「刪除」、「往上/下一階」應正常運作。

### B. 手機端拖曳停用 (Mobile Drag-and-Drop Disabled)
- [ ] **卡片觸控移動**：手指按住卡片並立即移動，不應觸發拖曳效果（卡片應留在原處，頁面可正常捲動）。
- [ ] **排序功能封鎖**：確認手機端無法透過觸控改變任何任務順序。

### C. 集中式鍵盤保護 (Centralized Keyboard Protection)
- [ ] **看板行內編輯**：進入卡片標題編輯模式，連續按 `Space`（空白鍵），不應觸發任務選取或拖曳。
- [ ] **待辦清單編輯**：在 Checklist 編輯時輸入含空白的文字並按 `Enter`，應正常儲存而非觸發 dnd-kit 的放下行為。
- [ ] **新增任務輸入**：在看板下方的「快速新增任務」輸入框打字，確認 `Space` 鍵不會誤觸。

## 2. 極端情境測試 (Extreme Edge Cases)

### E1. 手機端：長按與移動的邊界
- **情境**：按住任務 400ms（尚未觸發選單）時突然移動手指。
- **預期結果**：系統應判定為捲動意圖，**不應**彈出選單，也**不應**啟動拖曳。

### E2. 手機端：快速連續點擊
- **情境**：在 1 秒內快速點擊同一個任務 5 次。
- **預期結果**：系統應視為多次單擊（進入編輯模式），**不應**因計時器累積而錯誤彈出選單。

### E3. 桌面端：語音輸入結果
- **情境**：使用語音輸入一段長句（包含多個空白鍵），結果自動填入 Input。
- **預期結果**：SmartKeyboardSensor 應持續過濾，不應因大量字元湧入導致 dnd-kit 誤判。

### E4. 桌面端：輸入法 (IME) 組合期間
- **情境**：使用中文輸入法，在選字過程中按 `Space` 或 `Enter`。
- **預期結果**：事件應被 SmartKeyboardSensor 攔截，不應觸發背景任務的拖曳動作。

### E5. UI 邊界：螢幕邊緣觸發
- **情境**：在靠近螢幕最右側或最底部的任務上觸發長按。
- **預期結果**：`GlobalContextMenu` 應根據剩餘空間自動調整位置（menuX/menuY 計算），不應超出視窗範圍。

---

## 3. 負面測試 (Negative Testing)

- [ ] **非目標元素長按**：在看板空白背景處長按，不應彈出任務選單。
- [ ] **唯讀/鎖定狀態測試**：若任務處於鎖定狀態，長按選單內的敏感操作（如刪除）是否仍有適當提示。
